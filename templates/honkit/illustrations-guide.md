# Tutorial Illustrations Guide

This guide helps you select relevant, engaging illustrations for tutorial introductions and conclusions.

## xkcd Comics

xkcd comics are perfect for technical tutorials—relatable, humorous, and licensed under Creative Commons Attribution-NonCommercial 2.5.

### How to Use xkcd

**Format**:
```markdown
![xkcd comic title](https://imgs.xkcd.com/comics/image_name.png)

*[xkcd #NUMBER](https://xkcd.com/NUMBER/): "Comic Title" by Randall Munroe (CC BY-NC 2.5)*
```

**Attribution Requirements**:
- Always include the comic number and link to xkcd.com
- Credit Randall Munroe
- Mention the CC BY-NC 2.5 license
- Include the alt text as image alt attribute when possible

## Alternative Illustration Sources

### 1. **unDraw** (https://undraw.co/)
- **License**: Open source, free for any use
- **Style**: Modern, customizable SVG illustrations
- **Good for**: Generic tech concepts, team collaboration, development scenes
- **How to use**: Search by keyword, customize colors, download SVG
- **No attribution required** (but appreciated)

**Example**:
```markdown
![Developer illustration](https://undraw.co/illustrations/developer.svg)
```

### 2. **Unsplash** (https://unsplash.com/)
- **License**: Free to use (Unsplash License)
- **Style**: High-quality photography
- **Good for**: Abstract concepts, workspace photos, technology imagery
- **How to use**: Search, download, credit photographer
- **Attribution**: "Photo by [Name] on Unsplash"

**Example**:
```markdown
![Code on screen](https://images.unsplash.com/photo-1461749280684-dccba630e2f6)

*Photo by [Ilya Pavlov](https://unsplash.com/@ilyapavlov) on Unsplash*
```

### 3. **Pexels** (https://www.pexels.com/)
- **License**: Free to use (Pexels License)
- **Style**: Stock photography and videos
- **Good for**: Tech concepts, coding scenes, professional imagery
- **How to use**: Search, download, credit photographer
- **Attribution**: "Photo by [Name] from Pexels"

### 4. **Feather Icons** (https://feathericons.com/)
- **License**: MIT (free)
- **Style**: Simple, clean SVG icons

- **No attribution required**

### 5. **Simple Icons** (https://simpleicons.org/)
- **License**: CC0 (public domain)

- **No attribution required**

## Selection Guidelines

### For Introduction:
- **Purpose**: Set tone, acknowledge learning journey, make tutorial relatable
- **Good choices**:
  - Comics about learning something new (#1053)
  - Humor about the topic's complexity
  - Relatable developer experiences

### For Conclusion:
- **Purpose**: Celebrate achievement, validate effort, inspire next steps
- **Good choices**:
  - Comics about mastery or power gained (#208, #353)
  - Humor about newfound capabilities
  - Validation of time spent learning (#1205)

### Matching Tutorial Topics

**Web Development**:
- Standards (#927), APIs (#1481), JavaScript quirks

**Security/Authentication**:
- SQL Injection (#327), Password Strength (#936)

**DevOps/Automation**:
- Automation (#1319), CI/CD, Workflow (#1172)

**Machine Learning/AI**:
- Tasks (#1425), Data cleaning

**Testing**:
- Code Quality (#1513, #1833), Testing culture

**Version Control**:
- Git workflow (#1172), Merge conflicts

## Example Usage in Templates

### Introduction (index.md):
```markdown
# Welcome to Building Your First REST API

Learning to build APIs can feel overwhelming—there are so many concepts to juggle: HTTP methods, status codes, authentication, validation. But here's the thing: everyone starts somewhere.

![xkcd: Ten Thousand](https://imgs.xkcd.com/comics/ten_thousand.png)

*[xkcd #1053](https://xkcd.com/1053/): "Ten Thousand" by Randall Munroe (CC BY-NC 2.5)*

By the end of this tutorial, you'll understand not just how APIs work, but why they're designed the way they are.
```

### Conclusion (conclusion.md):
```markdown
## Final Thoughts

![xkcd: Automation](https://imgs.xkcd.com/comics/automation.png)

*[xkcd #1319](https://xkcd.com/1319/): "Automation" by Randall Munroe (CC BY-NC 2.5)*

Was it worth the time to learn all this? Absolutely. The API you just built isn't just working code—it's a foundation. Every project from here gets faster, cleaner, more robust.

You started this tutorial wondering how to build a REST API properly. You're leaving with the ability to make intentional architectural decisions and defend them with reasoning.
```

## Best Practices

### Do:
- ✅ Choose illustrations that genuinely relate to the tutorial content
- ✅ Provide proper attribution for all sources
- ✅ Test image URLs before adding (xkcd.com images are reliable)
- ✅ Consider accessibility - include descriptive alt text
- ✅ Keep illustrations relevant, not decorative

### Don't:
- ❌ Use images without proper licensing
- ❌ Skip attribution requirements
- ❌ Use too many images (one in intro, one in conclusion is enough)
- ❌ Choose images that might confuse or distract from content
- ❌ Use images that might become outdated or broken

### Accessibility:
Always include descriptive alt text:
```markdown
![Illustration showing a developer automating tasks with the caption about time spent vs saved](...)
```
